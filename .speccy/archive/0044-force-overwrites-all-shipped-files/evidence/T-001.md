### Scenario 1 — `.claude/agents/reviewer-business.md` is overwritten under `--force` (CHK-001)

<red>
Before the carve-out is removed, the test
`t002_claude_reviewer_agent_files_preserve_user_edits_under_force`
asserts the inverse property (sentinel survives `--force`). That
test was renamed and inverted to
`t002_claude_reviewer_agent_files_overwrite_user_edits_under_force`,
asserting `!after.contains("sentinel-edit-12345")` and that stdout
contains `(!) overwritten ... .claude/agents/reviewer-business.md`.
Running the new test against the pre-edit binary (carve-out still
in place) would fail at the `!after.contains(...)` assertion because
the carve-out branch in `append_host_pack_items` maps existing
reviewer files to `Action::Unchanged` regardless of byte equality,
so the appended sentinel persists.
</red>

<green>

in `append_host_pack_items`, the file participates in
`classify_content`. The appended sentinel makes the file
byte-different from the planned bundle, so the classification is
`Action::Conflict`; under `--force` the file is overwritten.


</green>
