## CHK-004 — plan-architect renders for both hosts; advisory non-verdict contract; agent-sized build sequence

Three structural tests in `speccy-cli/tests/skill_packs.rs` drive the
real MiniJinja render pipeline (`render_host_pack`, the same engine
`just reeject` uses, walking the embedded `RESOURCES` bundle) for both
hosts:

- `plan_architect_renders_with_includes_expanded_both_hosts` — asserts
  both wrappers exist and contain no `{%` (all includes expanded).
- `plan_architect_body_has_no_review_verdict_marker_both_hosts` —
  asserts the rendered body contains no `<review` verdict-contract
  marker.
- `plan_architect_body_specifies_agent_sized_build_sequence_both_hosts`
  — asserts the body contains `agent-sized` and describes a
  build-sequence checklist.

### Red phase

Before `touch speccy-cli/src/embedded.rs && just reeject` re-embedded
the new wrapper files, the render pipeline did not include
`plan-architect.{md,toml}`. The first reeject run reported
`0 created` and the ejected files were absent
(`ls .claude/agents/plan-architect.md` → no such file). A test run at
that point would panic in `rendered_plan_architect_body` with
"rendered host pack must include `.../plan-architect.{suffix}` after
T-003".

### Green phase

After the forced rebuild, both wrappers ejected
(`1 created` per host), and `cargo test --workspace` reported all
three new tests `ok`:

```
test plan_architect_body_specifies_agent_sized_build_sequence_both_hosts ... ok
test plan_architect_body_has_no_review_verdict_marker_both_hosts ... ok
test plan_architect_renders_with_includes_expanded_both_hosts ... ok
```

Rendered-content spot checks: `grep -c "{%"` → 0 for both files;
`grep -c "<review"` → 0 for both files.
