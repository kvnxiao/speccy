Section-heading detection ignores block bodies and fenced code, so a valid VET.md is mis-partitioned and spuriously rejected.

`collect_invocation_headings` (`speccy-core/src/parse/vet_xml/mod.rs:588`) runs `invocation_heading_regex()` (`:234`, a `(?m)^##\s+Invocation\s+\d+\s+(?:—|-)\s+\S+\s*$` match) over the entire `source` with no awareness of which byte ranges fall inside a block body or a fenced code block. This is asymmetric with `reject_unknown_block_tags` (`:336`), which is correctly passed `code_fence_ranges` and skips fenced lines.

Consequence: any `<drift-review>` / `<holistic-fix>` / `<simplifier-*>` / `<gate>` body that contains a line shaped like `## Invocation N — <token>` — whether free prose or inside a ``` fence — is treated as a real section heading. `partition_into_sections` then splits the surrounding section and `validate_section` rejects the document with a misleading "invocation N section has no terminal `<gate>` block".

Reproduced against the parser: a `<drift-review>` body containing the prose line `## Invocation 2 — 2026-05-21T19:00:00Z`, followed by a normal terminal `<gate>`, parses to `Err(MalformedMarker { reason: "invocation 1 section has no terminal `<gate>` block" })` instead of a one-section `VetDoc`. The identical content wrapped in a ``` fence fails the same way.

Reachability: vet block bodies are free-form prose authored by reviewer/implementer/simplifier personas, and `vet-phases.md` itself documents the `## Invocation (\d+)` heading convention and instructs pasting prior blocks/context verbatim into prompts — so a body quoting the file's own section structure is plausible. Blast radius is bounded (the tolerant `next.rs` gate scanner is unaffected and the parser is wired into no read path in T-003), but per-section partition correctness and "valid documents parse to the correct structure" are this task's core contract. Fix by passing `collect_code_fence_byte_ranges(source)` (and ideally the assembled block body spans) into `collect_invocation_headings` and filtering heading matches inside a fenced range or block body. A regression test with a heading-shaped line inside a block body and inside a fence should accompany the fix.
