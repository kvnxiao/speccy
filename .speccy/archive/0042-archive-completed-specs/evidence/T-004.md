# Evidence for SPEC-0042 T-004

This file is the paper-trail proof for T-004 (detect
supersession-chain orphan and emit warning on archive).

The three CHKs under REQ-008 (CHK-020, CHK-021, CHK-022) are
labelled `demonstrated` because they are exercised by the new
`speccy-cli/tests/archive_warnings.rs` test binary, which builds
fixture workspaces and runs the real `speccy archive --json`
command end-to-end, parsing the resulting JSON and asserting on
the stderr warning lines.

The pre-edit "red" form is the absence of any warning emission
path: before this task, `ArchiveOutcome` had no `orphan_warnings`
field, `ArchiveReceipt::from_outcome` always produced
`warnings: []`, and no stderr warning was written. The post-edit
"green" form is the passing test suite below, which now sees a
populated `warnings` array in the sole-declarer scenario and an
empty array in the natural and multi-declarer scenarios.

## CHK roll call (REQ-008)

- CHK-020 — demonstrated. Test
  `archive_emits_orphan_warning_when_sole_declarer` builds a
  workspace with SPEC-0019 (`status: superseded`) and SPEC-0021
  (`supersedes: [SPEC-0019]`) as the sole declarer, then runs
  `speccy archive SPEC-0021 --json`. Asserts: exit 0, stderr
  contains both `SPEC-0019` and `SPEC-0021` along with the
  `warning:` prefix, and stdout JSON's `warnings` array equals
  `[{"spec":"SPEC-0019","reason":"orphaned-supersession"}]`.

- CHK-021 — demonstrated. Test
  `archive_older_spec_in_pair_emits_no_warning` uses the same
  fixture but archives SPEC-0019 (the older, superseded spec —
  the natural case). Asserts: exit 0, stderr carries no
  `warning:` substring, JSON `warnings == []`.

- CHK-022 — demonstrated. Test
  `archive_multi_declarer_emits_no_warning` builds a workspace
  where SPEC-0021 and SPEC-0022 both declare
  `supersedes: [SPEC-0019]`, then archives SPEC-0021. Asserts:
  exit 0, stderr carries no warning, JSON `warnings == []`
  (SPEC-0022 still explains SPEC-0019's superseded status).

## Supplemental tests (not separate CHKs)

- `archive_with_empty_supersedes_emits_no_warning` — guards the
  empty-`supersedes` short-circuit in the detector. Archives a
  solo SPEC-0030 with `supersedes: []` and asserts the JSON
  `warnings` is `[]`.
- The pure unit test `orphan_target_not_in_active_set_returns_empty`
  guards the "target already archived" condition (REQ-008
  step 2(a) check on active membership): if the orphan target
  has itself been moved out of `.speccy/specs/`, archiving the
  later spec does not emit a warning.
- `orphan_target_not_superseded_returns_empty` guards REQ-008
  step 2(b): a target with `status: implemented` (rather than
  `superseded`) does not surface as an orphan.
